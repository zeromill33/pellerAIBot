# 附录 I：运营 SOP（落地后推送版）

## I1. 目标

- 运营可快速“输入链接 → 产出报告 → 先落地 →（可选运营校验）→ 推送 Channel”
- 允许“重复生成”用于纠正不满意输出，但必须受限流保护额度与稳定性

## I2. 操作步骤

1) 运营输入 Polymarket 链接（数量不固定）  
2) 系统拉取 Polymarket 数据（Gamma/CLOB）  
3) 系统执行 Tavily 多车道检索（默认 A/B/C；D 条件触发且多类型查询）  
4) LLM 生成 Report v1 JSON  
5) Validator 校验：  
   - 通过 → 报告与文案落地保存 → 按发布策略处理  
   - 失败 → 返回失败原因（缺反方证据/缺规则原文/失败路径太泛/喊单等）  

6) 发布策略（配置化二选一）：  
   - 自动发布：直接推送到 Channel  
   - 运营确认：运营审核草稿后再推送  

7) 运营可点击“重新生成”：  
   - 同事件 **5 分钟内最多 1 次**  
   - 同事件 **1 小时最多 5 次**  
   - 规则从配置文件读取，可调整  

## I3. 运营抽查要点（建议 30 秒）

- 结算规则原文是否与市场页一致（是否被断章取义）
- 证据链接是否能打开且与 claim 对得上
- 失败路径是否具体（有没有“可观察信号”）
